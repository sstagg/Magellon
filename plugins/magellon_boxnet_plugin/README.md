# magellon_boxnet_plugin

External (broker-resident) BoxNet v2 CNN particle picker.

Category: `PARTICLE_PICKING` (code 3).
Backend id: `boxnet-picker`.
Cardinality: per-micrograph (one task per image).

## Third picker backend

Sits alongside `topaz` (CNN-based) and `template-picker` (FFT correlation)
under PARTICLE_PICKING. CoreService's dispatcher routes via
`TaskMessage.target_backend`. Operator picks which backend to use at
dispatch time; the React UI advertises the live ones via
`GET /plugins/capabilities`.

## Inputs

`BoxnetPickerInput` is plugin-owned (PE2-UI, 2026-05-12). Required: an
image path. The rest have sensible defaults:

| Field | Default | Notes |
|---|---|---|
| `image_path` | — required — | Absolute path to .mrc |
| `threshold` | 0.3 | Particle-channel probability cutoff in [0, 1] |
| `min_distance` | 14 | Minimum centre-to-centre spacing (preprocessed px) |
| `scale` | 8 | Downsample factor on the way in |
| `device` | `auto` | `cuda` / `cpu` / `auto` |
| `output_dir` | — | Override for the result JSON location |

## Outputs

- `particles.json` — list of `{center: [x, y], radius: px, score: 0..1}`
- `ParticlePickingOutput.particles_json_path` — the path on the data plane.
  Per ratified rule 1, the inline `picks` list is left empty.

## Weights

Drop `boxnet.pt` into `weights/`. The Warp pretrained models live at
http://boxnet.warpem.com/models; the same file the Cianfrocco-lab
`eval_micrograph` repo ships works (it's a graph-trace via onnx2torch
of the Warp BoxNet v2 TF model).

The Dockerfile COPYs `weights/` into the image; build with the `.pt`
already in place. For uv-method installs, `weights/` is found relative
to the package root at runtime.

## Provenance

Algorithm is vendored verbatim from
`Sandbox/particle_picking_boxnet/`. The `flexible_forward` graph-trace
in `plugin/algorithm.py` is auto-generated by onnx2torch and is **not**
to be hand-edited — see `Sandbox/particle_picking_boxnet/boxnet_pt.py`
for the validation story.
