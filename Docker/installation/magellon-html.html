Magellon Installation Guide v1.0

Important Information

This guide covers the Dockerized installation of Magellon. For non-Docker installations, please refer to the standard installation guide.

This documentation will guide you through installing and configuring Magellon using Docker. The Dockerized version simplifies dependency management and ensures consistent performance across different environments.

Install Docker

Docker Desktop is the recommended installation method as it provides a consistent experience across platforms.

  • Download and install Docker Desktop for your operating system
  • Verify installation by running docker --version in your terminal
Download the Demo Data

Access the demo dataset using Globus file transfer service.

  • Navigate to Globus File Manager
  • Log in or create a Globus account if needed
  • Transfer the demo files to your local system
Download Magellon Source Code

Clone the Magellon repository from GitHub to your local system.

BASH 
git clone https://github.com/sstagg/Magellon.git

Alternatively, you can download the ZIP archive from the GitHub repository.

Create Magellon Directories

Set up the required directory structure for Magellon operation.

Directory Structure

Magellon requires three main directories for proper operation:

  • Home Directory: Where Magellon stores generated files

    Example: /Users/username/magellon/home

  • GPFS Directory: For data imports (mapped to "/gpfs" internally)

    Example: /Users/username/magellon/gpfs

  • Jobs Directory: For temporary file storage

    Example: /Users/username/magellon/jobs

Configure Environment Settings

Set up the environment configuration file for Magellon.

  • Navigate to the Docker subdirectory in the Magellon source
  • Copy the example configuration file
    $ cp .env.example .env
  • Edit the .env file to configure your Magellon directories 
    MAGELLON_HOME_PATH=/Users/username/magellon/home
MAGELLON_GPFS_PATH=/Users/username/magellon/gpfs
MAGELLON_JOBS_PATH=/Users/username/magellon/jobs
Start Magellon Containers

Launch the Docker containers for Magellon.

  • Execute the start script from the Docker directory
    $ bash start.sh
  • Wait for container initialization (may take longer on first run)
  • For subsequent starts, you can use:
    $ docker compose --profile default up -d

    Or start the containers through Docker Desktop interface

First Run Performance

The first time you execute start.sh, it will download and initialize all supporting containers. This process may take some time depending on your internet connection. Subsequent startups will be much faster.

Import Demo Data

Import the downloaded demo data into Magellon.

  • Move the Globus demo data to your Magellon GPFS directory
  • Access the Magellon frontend at http://localhost:8080/en/panel/images
  • Click "Login" in the interface
  • Click the three stacked lines icon in the top left corner
  • Select "Import" and navigate to the "Magellon import" tab
  • Double-click the 24dec03a directory icon
  • Double-click the session.json file
  • Click the "IMPORT DATA" button to begin import
Import Processing Time

The data import process may take 5-60 minutes depending on your system resources. On Linux systems with compatible GPUs, Magellon will perform frame alignment which may extend processing time.
